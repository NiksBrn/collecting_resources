# Улучшения алгоритма AliceBot

## 1. Динамический бюджет исследования

**Постановка:** не ограничиваться фиксированной половиной еды на исследование. Отслеживать остаток еды и сколько необходимо на возврат домой. Как только достигаем лимита — идём домой.

**Реализация:**
- На каждом шаге перед перемещением вычислять `returnCost` — длину кратчайшего пути до выхода из текущей позиции (BFS по посещённым).
- Условие продолжения: `game.food() > returnCost`.
- Вместо `budget = M / 2` цикл `while (food > shortestReturnPath(current))`.

**Плюс:** бот использует еду максимально эффективно — исследует ровно столько, сколько позволяет запас с учётом реального расстояния до выхода.

## 2. Оптимальный сбор ресурсов при возврате

**Постановка:** при жадном сборе по пути домой может закончиться еда раньше, чем мы заберём самые ценные ресурсы. Нужно заранее оценить, какие ресурсы собирать в каких комнатах, а также рассмотреть альтернативные пути домой.

**Реализация:**

*Оценка ресурсов на маршруте:*
- Перед возвратом найти все пути домой, на которые хватает еды.
- Для каждого пути подсчитать суммарную ценность собираемых ресурсов с учётом стоимости.
- Выбрать путь с максимальной ценностью.

*Планирование сбора на маршруте:*
- Составить список (комната, тип ресурса, ценность).
- Отсортировать по `ценность`.
- Жадно выбирать с конца пока хватает `extra` еды.

*Альтернативные пути:*
- BFS/DFS для поиска всех путей длиной `<= food` из текущей позиции до комнаты 0.
- Для каждого пути считать суммарную ценность ресурсов которые можно собрать.
- Выбирать путь с наибольшей ценностью.

## 3. Сбор ресурсов выше среднего по пути к цели

**Постановка:** при движении к следующей непосещённой комнате бот проходит через уже посещённые. В них могут оставаться несобранные ресурсы (если типов несколько). Собирать их если ценность выше среднего — может быть выгодно.

**Реализация:**

*Подсчёт среднего:*
- Отслеживать `totalValue / totalCollected` по всем собранным ресурсам.
- Или считать среднее по первым сборам (бесплатным) как базовую планку.

*Сбор по пути:*
- При движении через промежуточную комнату проверять: есть ли несобранный ресурс с `count * valueOf(t) > avgValue`.
- Если да и есть `extra` еда — собирать.
- `extra` здесь = `food - returnCost(current)`
